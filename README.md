# SPI_with_FPGA



##  Overview

This project implements a **Serial Peripheral Interface (SPI) Slave** in FPGA, designed to communicate with an **ESP32 (SPI Master)**.

The goal is to build the SPI protocol **from scratch at RTL level**, without using vendor IP cores, to deeply understand:

* Serial communication protocols
* Timing synchronization
* Shift register-based data transfer
* Hardware design using Verilog

---

## What is SPI?

SPI (**Serial Peripheral Interface**) is a synchronous communication protocol where a **Master device controls data flow** and communicates with one or more **Slave devices** using a shared clock.

###  SPI Signals

| Signal | Description                      |
| ------ | -------------------------------- |
| SCLK   | Serial Clock generated by Master |
| MOSI   | Master Out Slave In              |
| MISO   | Master In Slave Out              |
| CS     | Chip Select (Active LOW)         |

## 🎯 Project Objectives

* Design and implement an SPI Slave in FPGA
* Receive serial data from ESP32 via MOSI
* Convert serial data into parallel data
* Generate a "data ready" signal after full byte reception
* Understand real-time hardware timing

---

## ⚙️ System Architecture

```
ESP32 (Master)
     │
     │ MOSI, SCLK, CS
     ▼
FPGA (SPI Slave)
     │
     ├── Shift Register
     ├── Bit Counter
     └── Data Ready Logic
```

---

## 🔄 Working Principle

1. Master (ESP32) pulls **CS LOW** to select FPGA
2. Master generates **SCLK**
3. On each clock edge:

   * FPGA reads **MOSI bit**
   * Shifts it into register
4. After 8 clock cycles:

   * A full byte is received
   * `data_ready` signal is asserted

---

## 🧩 RTL Design (Core Concept)

Shift Register Logic:

```verilog
shift_reg <= {shift_reg[6:0], mosi};
```

Bit Counter:

```verilog
bit_count <= bit_count + 1;
```
